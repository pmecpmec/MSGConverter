# Checklist Voordat je Inlevert
## Pre-Submission Checklist voor Deliverables

**Gebruik deze checklist voor ELKE deliverable die je inlevert!**

---

## Quick Check (Doe dit ALTIJD!)

```
□ Document is compleet en alle secties zijn ingevuld
□ Spelling & grammatica gecontroleerd
□ Formatting is consistent (headers, bullets, etc.)
□ Authenticiteitsverklaring is toegevoegd onderaan document
□ Juiste AIAS-niveau is vermeld in authenticiteitsverklaring
□ Datum, naam en project info zijn up-to-date
□ Alle referenties en links werken
□ Bestanden zijn opgeslagen in de juiste map
```

---

## AI-Transparantie Check

### Voor ALLE Deliverables

```
□ AI-gebruik is gedocumenteerd in `docs/ai/AI-GEBRUIK.md`
□ Juiste authenticiteitsverklaring gekopieerd uit `docs/ai/AI-AUTHENTICITEITSVERKLARINGEN.md`
□ AIAS-niveau klopt met daadwerkelijk gebruik
□ <<naam tool>> en <<activiteit>> zijn ingevuld (niet vergeten!)
□ Geen generieke placeholder tekst zoals "Cursor AI" zonder specificatie
```

### Niveau Check

**Niveau 2 (AI Exploratie)**
```
□ Document bevat GEEN AI-gegenereerde tekst (alleen brainstorm/planning output)
□ Verklaring zegt expliciet: "geen AI-gegenereerde inhoud"
□ Alle inhoud is door jou geschreven op basis van AI-ideeën
```

**Niveau 3 (AI Samenwerking)**
```
□ AI-suggesties zijn gereviewed en aangepast
□ Project-specifieke context is toegevoegd
□ Business logic/inhoud is van jou
□ Je kunt alles uitleggen tijdens assessment
```

---

## Document-Specifieke Checks

### Projectdefinitie Documenten

```
□ Context & achtergrond: Babcock Schiphol specifiek
□ Probleemstelling: Gebaseerd op echte situatie
□ Doelstellingen: SMART geformuleerd
□ Scope: Duidelijk afgebakend, in overleg met stakeholders
□ Stakeholders: Echte namen/rollen (of geanonimiseerd)
□ Authenticiteitsverklaring: Niveau 2 of 3
```

### Plan van Aanpak

```
□ Methodiek: Keuze onderbouwd (Agile/Scrum/Waterfall)
□ Planning: Realistische tijdsinschattingen
□ Risicoanalyse: Project-specifieke risico's
□ Mitigaties: Concrete acties per risico
□ Deliverables: Duidelijke acceptatiecriteria
□ Authenticiteitsverklaring: Niveau 2
```

### Onderzoek Documenten

```
□ Literatuur/bronnen: Correct geciteerd
□ Alternatieven: Minimaal 2-3 opties vergeleken
□ Keuze: Onderbouwd met rationale
□ POC/Experimenten: Resultaten gedocumenteerd
□ Conclusie: Duidelijke aanbeveling
□ Authenticiteitsverklaring: Niveau 3
```

### Technisch Ontwerp

```
□ Architectuur diagram: Duidelijk en volledig
□ UML diagrammen: Correct syntax, up-to-date
□ Class design: SOLID principles toegepast
□ API specificaties: Complete endpoints, request/response
□ Design decisions: Gedocumenteerd met rationale
□ Authenticiteitsverklaring: Niveau 3
```

### Source Code

```
□ Code compileert/draait zonder errors
□ Type hints overal (Python) of types (TypeScript)
□ Docstrings/comments voor alle publieke functies
□ Business rules compliance gecontroleerd
□ Geen hardcoded credentials
□ .env.example up-to-date (zonder echte values)
□ Unit tests >80% coverage
□ Linter warnings = 0
□ Authenticiteitsverklaring: Niveau 3 (in README of docs/)
```

### Test Documentatie

```
□ Test cases: Duidelijk beschreven
□ Expected outcomes: Gedefinieerd
□ Test data: Gedocumenteerd
□ Coverage report: Inclusief percentage
□ Edge cases: Getest en gedocumenteerd
□ Authenticiteitsverklaring: Niveau 3
```

---

## Privacy & Security Check

```
□ Geen echte Maximo credentials in code/docs
□ Geen echte vliegtuigregistraties in voorbeelden
□ Geen stakeholder persoonsgegevens (of geanonimiseerd)
□ .env files in .gitignore
□ Geen API keys in Git history
□ Mock/test data gebruikt voor voorbeelden
```

---

## Kwaliteit Check

### Professionaliteit

```
□ Taal: Professioneel en formeel (geen spreektaal)
□ Structuur: Logische opbouw met headers
□ Leesbaarheid: Duidelijke zinnen, geen jargon zonder uitleg
□ Visual aids: Diagrammen, tabellen, bullets waar nuttig
□ Bronvermelding: Alle bronnen correct vermeld
```

### Technische Correctheid

```
□ Facts gecontroleerd (API docs, specs, etc.)
□ Code voorbeelden: Getest en werkend
□ Commando's: Getest op Windows/Linux
□ Paths: Correct voor platform (/ vs \)
□ Versie nummers: Up-to-date en correct
```

---

## Competentie Aantoonbaarheid

### Analyseren
```
□ Requirements duidelijk gedefinieerd
□ Probleem gedecomponeerd in kleinere delen
□ Stakeholder input verwerkt
□ Context en achtergrond uitgelegd
```

### Adviseren
```
□ Alternatieven overwogen en vergeleken
□ Keuzes onderbouwd met rationale
□ Risico's geïdentificeerd
□ Aanbevelingen concreet en haalbaar
```

### Ontwerpen
```
□ Architectuur gedocumenteerd met diagrammen
□ Design patterns toegepast en uitgelegd
□ Data modellen gedefinieerd
□ API's gespecificeerd
```

### Realiseren
```
□ Code werkt en is getest
□ Tests hebben >80% coverage
□ Clean code principles toegepast
□ Deployment gedocumenteerd
```

### Manage & Control
```
□ Planning realistisch
□ Voortgang trackbaar
□ Risico's gemonitord
□ Kwaliteit geborgd
```

---

## Voor Inleveren (Final Check)

### Dag voor Inleveren

```
□ Alle bovenstaande checks gedaan
□ Document opgeslagen in correcte format (PDF/MD/etc.)
□ Bestandsnaam volgens conventie: `[deliverable]-[datum]-[naam].pdf`
□ Backup gemaakt (Git commit + push)
□ Email/submission platform getest
```

### Zelf-Review

```
□ Document doorgelezen alsof je het voor het eerst ziet
□ Typfouten/spelfouten gecorrigeerd
□ Screenshots/images correct ingevoegd
□ Appendices toegevoegd waar nodig
□ Inhoudsopgave up-to-date (indien van toepassing)
```

### Peer Review (Optioneel maar Aanbevolen)

```
□ Vraag medestudent/collega om review
□ Feedback verwerkt
□ Onduidelijkheden opgehelderd
```

## Red Flags (STOP en Fix!)

**Als één van deze van toepassing is, NIET inleveren:**

```
 Authenticiteitsverklaring ontbreekt
 Echte passwords/credentials in code of docs
 "Lorem ipsum" placeholder tekst nog aanwezig
 <<placeholder>> markeringen nog zichtbaar
 Code draait niet of heeft syntax errors
 Tests falen
 Linter warnings > 10
 Geen bronvermelding bij citaten
 Plagiarismcheck niet gedaan
 Deadline gemist zonder communicatie
```

De daadwerkelijke probleemstelling, doelen, scope en stakeholders zijn gebaseerd op mijn eigen analyse van de situatie bij Babcock Schiphol. Ik draag de volledige verantwoordelijkheid voor de inhoud van dit werk.

```markdown
*AIAS Niveau: 2 - AI Exploratie*
```

## 🆘 Hulp Nodig?

**Voor je inlevert, vraag jezelf af:**

1. **Begrijp ik alles in dit document?**
 - Ja → Ga door
 - Nee → Review nogmaals of vraag begeleider

2. **Kan ik dit uitleggen tijdens assessment?**
 - Ja → Ga door
 - Nee → Oefen presentatie, maak notities

3. **Ben ik trots op dit werk?**
 - Ja → Perfect, lever in!
 - Nee → Neem de tijd om het te verbeteren
